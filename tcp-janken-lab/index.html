TCPじゃんけん
ネットワーク対戦ラボ

2台のPCまたは2つのブラウザでじゃんけん対戦をしながら、 SYN / SYN-ACK / ACK / DATA / FIN / RST の流れを体験できる Python 製ネットワーク教材です。

学べること

  • TCPの接続開始を、SYN / SYN-ACK / ACK の順番で体験します。
  • DATAパケットとしてじゃんけんの手を送り、相手に届く様子を見ます。
  • FINによる切断と、順序が崩れたときのRSTを教材上で確認します。
  • サーバー側とクライアント側の役割分担を、同じUIで比較できます。

実行内容

  • ローカルHTTPサーバーを起動し、ブラウザ画面を配信します。
  • WebSocketサーバーを起動し、2人の操作を中継します。
  • LAN内の相手が接続できるよう、HTTP/WSポートで待ち受けます。
  • IPアドレス判定はOSのローカル名前解決を使い、外部接続は行いません。

使い方

  1. Python 3.10以降を用意します。
  2. janken.py と requirements.txt を同じフォルダに置きます。
  3. 依存ライブラリをインストールします。
python -m pip install -r requirements.txt
  1. 待ち受け側のPCでサーバーモードを起動します。
python janken.py --server
  1. もう一方のPCでクライアントモードを起動し、画面の案内に従って接続します。
python janken.py --client

1台のPCで試す場合は、別々のターミナルでサーバーモードとクライアントモードを起動して動作確認できます。

通信について

  • Google Fontsや外部CDNは読み込みません。
  • IP判定のために外部サーバーへ接続する処理はありません。
  • 教材の動作には、同一PCまたは同一LAN内でのHTTP/WebSocket通信を使います。
  • 必要に応じて --http-port と --ws-port でポート番号を変更できます。

利用上の注意

  • 起動中は指定ポートで待ち受けるため、信頼できるLAN内の演習で使ってください。
  • 公共Wi-Fiや不特定多数が接続できるネットワークでは起動しないでください。
  • Windowsファイアウォールの確認画面が出た場合は、演習環境に合わせて許可範囲を選んでください。
  • 実際のTCP実装ではなく、TCPの考え方を学ぶための教材シミュレーションです。
