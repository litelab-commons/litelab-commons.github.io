# サイバーすごろく / Cyber Sugoroku

サイバーセキュリティ教育向けのネットワーク対応すごろくゲームです。
ホスト画面をプロジェクターに投影し、参加者は各自のブラウザから参加します。

ランサムウェア、フィッシング、バックアップ、MFA、EDR、CSIRT、BCP などをゲーム内イベントとして扱い、遊びながら「備え」と「被害軽減」の意味を体感できます。

## ファイル構成

- `server.py` ... FastAPI + WebSocket サーバ、ゲームロジック、盤面設定
- `host.html` ... ホスト画面、盤面、進行モード、最終ランキング
- `client.html` ... 参加者画面、サイコロ、横スクロールのルート表示、効果音
- `start.bat` ... Windows 用起動バッチ
- `requirements.txt` ... Python 依存パッケージ

## 起動方法（Windows）

1. Python 3.10 以上をインストール
2. `start.bat` をダブルクリック
3. 初回起動時は仮想環境作成と依存パッケージのインストールが実行されます
4. コマンドプロンプトに表示される URL を確認します
   - ホスト画面: `http://<PCのIP>:8000/`
   - 参加者画面: `http://<PCのIP>:8000/play`
   - ローカル確認: `http://localhost:8000/`
5. ホスト PC で `http://localhost:8000/` を開き、プロジェクターに投影します
6. 参加者は同じ LAN 内のブラウザから `http://<PCのIP>:8000/play` にアクセスします
7. 参加者が名前を入力したら、ホスト画面の「ゲーム開始」を押します

## ゲーム概要

- 30 マスのサイバー演習ボードを進みます
- 参加者は各ラウンドに 1 回サイコロを振ります
- 振った参加者はすぐに移動します
- ゴールは 29 マス目です
- ゴールを超えた場合は跳ね返ります
  - 例: 27 マス目で 5 を出すと 26 マス目へ戻ります
- 20 ラウンドでゲーム終了です
- 各ラウンド終了時、未ゴール参加者は継続ボーナスとして `+3pt` を得ます

## 勝敗とランキング

最終結果では、ホスト画面が `FINAL RESULT` に切り替わります。

- 着順ランキング: ぴったりゴールした順
- SECURITY PT ランキング: 最終 pt が高い順

20 ラウンド終了時点で未ゴールの参加者も、pt ランキングには表示されます。

## pt と防御アイコン

参加者は初期値 `100pt` を持っています。
良い施策で pt が増え、インシデントで pt が減ります。

一部の施策は防御アイコンとして残り、後の被害を軽減します。

- `💾` バックアップ ... ランサムウェア被害を軽減
- `🛡️` MFA ... フィッシング被害を軽減
- `👁️` EDR ... 標的型攻撃、ランサムウェア被害を軽減
- `🚨` CSIRT ... ランサムウェア被害を軽減
- `🏢` BCP ... ランサムウェア被害を軽減

## 代表的なイベント

- ランサムウェア感染
  - 発動者はスタートへ戻り、`-50pt`
  - 他の未ゴール参加者も被害を受けます
  - 防御アイコンがある場合は被害が軽減されます
- フィッシングメール
  - 通常は戻る + pt 減少
  - MFA 済みなら被害軽減
- 標的型攻撃
  - 通常は大きく戻る + pt 減少
  - EDR 済みなら被害軽減
- セキュリティ研修 / 情シス相談
  - 追加で 1〜3 マス進みます
- CSIRT 訓練 / 経営層へ報告
  - 1 回休み系イベントです

## ホスト用進行モード

ゲーム開始前は、ホスト画面下部に参加 URL と QR コードが表示されます。

ゲーム開始後は、下部が進行パネルに切り替わります。

- 読み上げ
- 解説
- 問いかけ

イベントに応じて、進行役がそのまま読める短い説明が表示されます。

## 参加者画面

参加者画面には次の要素があります。

- 自分の位置、pt、防御アイコン
- 横スクロールするルートパネル
- サイコロボタン
- 他参加者の状態
- イベントメッセージ
- イベント種別ごとの効果音

効果音はブラウザの制約により、参加ボタンやサイコロボタンなどのユーザー操作後に有効になります。

## カスタマイズ

`server.py` 上部の `SQUARES` を編集すると、盤面イベントを変更できます。

あわせて `BUDGET_EFFECTS` を編集すると、各イベントの pt 増減を変更できます。

ホスト進行モードの文言は `host.html` 内の `FACILITATOR_NOTES` で編集できます。

## ファイアウォール設定

Windows ファイアウォールでポート `8000` の受信を許可してください。
初回起動時にダイアログが出た場合は「アクセスを許可する」を選択します。

## ネットワーク要件

- ホスト PC と参加者端末が同一 LAN にあること
- ホスト PC の IP アドレスは `ipconfig` で確認できます
